個人運用でも散らかりやすいセキュリティ情報を、一つの画面で扱えるように整理した PWA。 記録、監視、バックアップ、暗号化までを軽量な静的構成でまとめています。

Summary

この作品で見せたいこと

  • 情報量の多い画面でも、使う順番を崩さずに整理できること
  • 静的サイトベースでも PWA と同期機能を積み上げられること
  • 個人ユースのセキュリティ管理を実用レベルまで持っていく視点

Overview

課題と設計

IT資産台帳、脆弱性管理、インシデント記録、監査チェック、パスワード管理は、 本来バラバラの場所に散らかりやすい領域です。そこで、個人運用でも継続しやすいように 「一画面で見渡せること」と「保存や復旧がしやすいこと」を優先して設計しました。

画面ごとの切り替えコストを減らしながら、重要情報だけは埋もれないように、 タブ構成、優先監視リスト、リスクマトリクス、JSONバックアップなどを組み合わせています。

公開サイト GitHub リポジトリ その他の制作物を見る

Highlights

主な機能

  • IT資産、脆弱性、インシデント、監査チェックを同一画面で管理
  • PBKDF2 + AES-GCM による暗号化ボルトでパスワード情報を保護
  • PWA対応により、スマホでの利用とホーム画面追加を想定
  • Netlify Functions + Blobs により、端末間同期の最小実装を追加
  • Service Worker のキャッシュ戦略を見直し、iPhone で古い画面を握る不具合を修正

Why It Matters

この作品が示す強み

この作品は、単に機能を増やすのではなく、情報量が多い実務画面をどう整理して、 どうやって日常運用に落とし込むかを形にしたものです。

UI整理、公開運用、同期、キャッシュ不具合の解消まで一連で扱っているため、 「作って終わりではなく、使われ続ける状態まで持っていく」姿勢がよく表れています。
